# QR Code Architecture Standard

## Rule

No QR code generated by Shepherd Kids may ever expose a database primary key or internal identifier. All QR codes must resolve through revocable, cryptographically secure random tokens.

## Scope

This rule applies to every QR feature in the platform, including:

- **Smart Labels** — per-child check-in label scanned by staff at pickup
- **Classroom QR** — per-room code scanned by volunteers to open Classroom View
- **Parent Pickup labels** — any future label artifact for parent-side scanning
- **Certificates** — printed or emailed achievement certificates with embedded QR
- **Parent Portal invitations** — QR links granting family portal access
- **Any future printed or shareable artifact** containing a QR code

## Why

Printed QR codes leave the building. They can be photographed, copied, laminated, cached, shared, or lost. A QR code containing a permanent database identifier (UUID, integer ID, slug) can never be invalidated without changing the underlying schema.

A revocable token means a single database field update immediately invalidates all previously printed copies of that QR code — no schema changes, no cascading updates, no data migration.

## Required Pattern

Every QR feature follows this pattern:

```
Database column:   some_qr_token  TEXT  DEFAULT encode(gen_random_bytes(18), 'hex')
QR code contains:  /feature/{some_qr_token}
Server resolves:   token → internal record → response
```

The token is the only public identifier. Internal primary keys remain private forever.

## Regeneration

Every token column must be capable of regeneration by updating the column value. When regenerated:

- The old token immediately becomes invalid
- Previously printed QR codes stop working
- No database relationships change
- No schema migration is required

## Audit Fields

Token columns should be accompanied by:

- `*_qr_created_at TIMESTAMPTZ DEFAULT NOW()` — when the token was first issued
- `*_qr_regenerated_by UUID NULL` — user ID of whoever last regenerated the token

## Current Implementations

| Feature | Token Column | Route |
|---|---|---|
| Smart Labels | `cm_checkin_records.qr_token` | `/dashboard/children-ministry/scan/[token]` |
| Classroom QR | `cm_checkin_rooms.classroom_qr_token` | `/classroom/[token]` |

## Never Do This

```
# Bad — exposes primary key
/classroom/123e4567-e89b-12d3-a456-426614174000

# Bad — exposes slug or sequential ID
/portal/smith-family
/certificate/1042

# Good — opaque revocable token
/classroom/a3f8c2e1b9d047f6c5a2
```
